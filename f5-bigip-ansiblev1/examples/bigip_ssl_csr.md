# bigip_ssl_csr


## Description

The `bigip_ssl_csr` module creates Certificate Signing Requests (CSRs) on F5 BIG-IP devices. It allows you to define subject attributes, key properties, and subject alternative names, then generates a CSR file that can be submitted to a Certificate Authority. This module is typically used alongside key and certificate modules to fully automate the SSL certificate lifecycle on BIG-IP.


## Parameters


| Parameter | Choices/Defaults | Comments |
| --- | --- | --- |
| **name** | **Type:** string<br>**Required:** yes | Name of the CSR object on the BIG-IP. |
| **partition** | **Default:** Common | Administrative partition where the CSR is created. |
| **common\_name** | **Type:** string<br>**Required:** yes | Common Name (CN) for the certificate, usually the FQDN. |
| **key** | **Type:** string<br>**Required:** yes | Name of the associated SSL key object on the BIG-IP. |
| **country** | **Type:** string | Country code (C) in the subject. |
| **state\_or\_province** | **Type:** string | State or province (ST) in the subject. |
| **locality** | **Type:** string | City or locality (L) in the subject. |
| **organization** | **Type:** string | Organization (O) in the subject. |
| **organizational\_unit** | **Type:** string | Organizational Unit (OU) in the subject. |
| **email\_address** | **Type:** string | Email address in the subject. |
| **subject\_alt\_name** | **Type:** list | List of Subject Alternative Names (DNS/IP) to include in the CSR. |
| **state** | **Choices:** present, absent | `present` creates or updates the CSR; `absent` removes it. |
| **provider** | **Type:** dictionary | Connection details for the BIG-IP device. |


## Return Values


| Key | Description |
| --- | --- |
| **name** | Name of the CSR object created or managed. |
| **csr\_text** | PEM-formatted CSR text generated by the BIG-IP. |
| **partition** | Partition where the CSR resides. |
| **state** | Final state of the CSR object. |


## Examples


```yaml
- name: Generate CSR for web application
  bigip_ssl_csr:
    name: "www_example_com.csr"
    partition: "Common"
    common_name: "www.example.com"
    key: "www_example_com.key"
    country: "US"
    state_or_province: "California"
    locality: "San Jose"
    organization: "Example Corp"
    organizational_unit: "IT"
    email_address: "admin@example.com"
    subject_alt_name:
      - "www.example.com"
      - "example.com"
    state: present
    provider: "{{ bip }}"
  delegate_to: localhost
```
